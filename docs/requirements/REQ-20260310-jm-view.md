# REQ-20260310-jm-view

## 需求摘要

一款以 JSON 为底层数据格式的 AI 原生结构化文档编辑器，提供类 Notion 的可视化编辑体验，普通用户可像使用 Word 一样输出结构化文档，LLM 天然直接可读。

---

## 背景与目标

**问题**：Markdown 是机器可读性与可视化编辑的折中产物，非原生结构化，LLM 需要额外解析。  
**目标**：以 JSON 作为唯一底层格式，通过可视化编辑器屏蔽 JSON 语法复杂度，同时保留 JSON 对 LLM 天然可读的优势。  
**用户**：普通用户（有基础记事本使用经验）+ 技术用户，两者兼顾。  
**市场定位**：AI 原生时代的结构化文档编辑器。

## 功能清单

| 编号 | 功能点 | 优先级 | 说明 |
|------|--------|--------|------|
| F01 | 三栏式布局 | 必须 | 左侧大纲（可收起）、中间主编辑区、右侧 JSON 原文（可收起） |
| F02 | JSON 树形大纲导航 | 必须 | 左侧面板显示文档 key 树形结构，点击跳转到对应卡片 |
| F03 | 卡片式可视化编辑 | 必须 | 每个 key-value 作为一张卡片，支持复制/粘贴/剪切/删除/拖动排序 |
| F04 | 类型角标显示 | 必须 | string=无角标，int=`#`，float=`#.`，bool=`T/F`，null=`null`，object=`{}`，array=`[]`；点击数字角标可在 int/float 间切换 |
| F05 | 斜杠命令系统 | 必须 | 输入 `/` 唤起命令菜单（可键盘/鼠标选择），覆盖文档结构/列表表格/媒体附件/文件操作/编辑/导航/界面等全类别命令，详见附录A |
| F06 | 层级输入规则 | 必须 | 换行=同级，Tab=子级（仅缩进一次），两次换行=子级的子级 |
| F07 | 特殊块渲染 | 必须 | 支持所有 `::` 前缀特殊块，详见附录B |
| F08 | 表格渲染 | 必须 | 对象数组→带列标题的表格；字符串数组→无序列表；列值连续相同时自动合并，支持开关切换原始/合并视图 |
| F09 | 右侧 JSON 原文双向编辑 | 必须 | 右侧面板可直接编辑 JSON，与可视化区实时双向同步 |
| F10 | 打开 / 新建 / 保存文件 | 必须 | 支持打开本地 JSON 文件；新建空白文档；Ctrl+S 保存 |
| F11 | 多标签页 | 必须 | 支持同时打开多个文件，参考 Notepad++ 风格的标签页管理 |
| F12 | 数据校验与自动修复 | 必须 | 检测缺闭合括号/多余逗号/缺少引号/单引号/未转义字符，修复前展示错误列表，用户确认后执行 |
| F13 | 草稿自动保存 + 崩溃恢复 | 必须 | 每 30 秒保存临时草稿到临时目录；启动时检测未保存草稿并提示恢复 |
| F14 | 搜索 / 替换 | 必须 | 普通搜索、正则表达式、标记计数、全文循环、选区查找、顺序/倒序 |
| F15 | 导出 Markdown | 必须 | 将 JSON 转为 Markdown；附件提取为独立文件导出到同目录 |
| F16 | 导入 Markdown | 必须 | 支持标题/段落/列表/表格/代码块/引用等基础元素；不兼容元素提示用户，转为纯文本正文导入 |
| F17 | 快捷键系统 | 必须 | 支持自定义；预设方案：Word / Notepad++ / VS Code；所有菜单项后显示对应快捷键 |
| F18 | 深色 / 浅色主题 | 必须 | 界面主题切换（深色/浅色）；文档渲染样式可选 |
| F19 | 大文件处理 | 必须 | 文件 > 5MB 时显示警告及加载进度条，支持取消 |
| F20 | 导出 HTML | 应有 | 导出为可独立访问的 HTML 文档（含内嵌样式） |
| F21 | 导出 PDF | 应有 | 通过 Tauri 调用系统打印/渲染能力导出 PDF |
| F22 | 命令行参数 | 应有 | 支持 `jm-view <file>` 打开指定文件；支持传入多个文件（多标签页打开） |
| F23 | 在线 Web 版 | 后续 | MVP 仅做桌面端 |
| F24 | 国际化 | 后续 | MVP 仅支持中文界面 |

## 技术规范

| 项目 | 说明 |
|------|------|
| 桌面端框架 | Tauri（Rust 后端 + Web 前端） |
| 前端 | Web 技术栈（由 architect 确定具体框架） |
| 附件存储 | Base64 嵌入 JSON（单文件便携，便携性 > 体积） |
| 数据格式 | 严格 JSON，特殊块使用 `::` 前缀 key |
| 撤销粒度 | 字符级撤销 |
| 富文本 | 不支持加粗/斜体/删除线等内联格式，保持纯文本 |

## 验收标准

- AC01: 打开合法 JSON 文件，左侧大纲正确渲染树形结构，中间区域以卡片方式展示每个 key-value
- AC02: 在可视化区修改任意值，右侧 JSON 原文实时同步；在右侧直接编辑 JSON，中间卡片区实时反映变化
- AC03: 输入 `/` 后唤起命令菜单，键盘/鼠标可选择命令，选中后正确插入对应特殊块
- AC04: 创建包含 `::code` 块的文档，保存后重新打开，代码块正确渲染（含语法高亮、行号）
- AC05: 对象数组渲染为带列标题的表格，连续相同列值自动合并，切换开关可还原原始视图
- AC06: 导出 Markdown，含图片附件的文档中图片以独立文件导出到同目录，路径引用正确
- AC07: 导入标准 Markdown 文件，标题/段落/列表/代码块/表格正确转换为 JSON 结构；不支持的元素给出提示
- AC08: 编辑中强制关闭应用，重启后提示检测到未保存草稿，恢复后内容与关闭前一致
- AC09: 右侧输入语法错误的 JSON，编辑器给出明确错误提示并提供自动修复选项，确认后执行
- AC10: 搜索支持正则表达式，匹配结果高亮，支持全文循环和顺序/倒序切换
- AC11: 深色/浅色主题切换后，左侧大纲/中间编辑区/右侧 JSON 原文三栏同步应用新主题
- AC12: 同时打开 3 个不同 JSON 文件，多标签页正确显示，切换标签内容独立不干扰
- AC13: 打开 > 5MB 的 JSON 文件时，显示加载进度条并提供取消按钮

## 附录 B — 特殊块（`::` 前缀）规范

| 特殊块 | 说明 | 必填字段 | 可选字段 |
|--------|------|----------|----------|
| `::meta` | 文档元数据 | — | title, author, created, tags |
| `::code` | 代码块 | lang, content | lineNumbers, highlight |
| `::quote` | 引用（同文件内 key 路径） | 路径字符串 | — |
| `::divider` | 分割线 | true | — |
| `::null` | 占位符 | true | — |
| `::todo` | 待办事项 | content, done | — |
| `::link` | 超链接 | text, url | — |
| `::img` | 图片（base64） | data, ext | name, width, height, thumb |
| `::file` | 通用附件 | data, ext | name |
| `::video` | 视频 | data, ext | name |
| `::audio` | 音频 | data, ext | name |
| `::LaTeX` | 数学公式 | content | — |
| `::Mermaid` | 流程图/图表 | content | — |
| `::callout` | 通用提示框 | 文本字符串 | — |
| `::note` | 笔记框（蓝色） | 文本字符串 | — |
| `::tip` | 技巧框（绿色） | 文本字符串 | — |
| `::warning` | 警告框（黄色/红色） | 文本字符串 | — |

特殊块可作为顶级 key（无标题）或嵌套在普通对象内（父 key 作为标题），由用户自由决定。
